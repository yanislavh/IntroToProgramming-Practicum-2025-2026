### Цикли
--------------------------------------------------
Теоретични задачи:
---
1. Каква е разликата между двата цикъла?

```cpp
#include <iostream>
int main() {
    for (int i = 0; i < 5; i++) {

        if (i == 2)
           break;

        std::cout << i;
    }
}
```
```cpp
#include <iostream>
int main() {
    for (int i = 0; i < 5; i++) {

        if (i == 2)
           continue;

        std::cout << i;
    }
}
```
<details>
 <summary>Обяснение: </summary>
    В първия цикъл се използва операторът break, който прекъсва изпълнението на целия цикъл, когато условието i == 2 стане вярно. Така програмата спира да повтаря цикъла и излиза от него напълно.

```cpp
Изход: 01
```

Във втория цикъл се използва continue, който не прекъсва цикъла, а само пропуска текущата итерация, когато i == 2. Цикълът продължава със следващата стойност на i.

```cpp
Изход: 0134
```

Следователно разликата е, че:

 - break спира цикъла напълно;

 - continue само прескача една итерация и продължава нататък.
</details>



2. Ще се отпечата ли „End“?
```cpp
#include <iostream>
int main() {
    int i = 0;
    while (i < 5) {
        continue;
        i++;
    }
    std::cout << "End";
}
```

<details>
 <summary>Обяснение: </summary>
В този пример операторът continue се изпълнява преди увеличаването на променливата i. Когато програмата стигне до continue, тя прескача останалия код в тялото на цикъла и веднага се връща в началото му, без да изпълни i++. Така стойността на i винаги остава 0 и условието i < 5 винаги е вярно.


В резултат цикълът се изпълнява <b>безкрайно</b> и програмата никога не стига до реда std::cout << "End";, затова „End“ няма да се отпечата.
</details>

3. Какво ще отпечата програмата?
```cpp
#include <iostream>
int main() {
    int n = 0;
    while (n = 3) {
        std::cout << n;
        if (--n == 0)
            break;
    }
}
```

<details>
 <summary>Обяснение: </summary>
 Програмата ще отпечатва безкрайно „3“, защото в условието се използва <b>присвояване</b> (=) вместо <b>сравнение</b> (==).  
</details>

4. Колко пъти ще се отпечата нещо?
```cpp
#include <iostream>
int main() {
    for (int i = 1; i <= 2; i++) {
        for (int j = 1; j <= 3; j++) {
            if (j == 2) break;
            std::cout << i << j << " ";
        }
    }
}
```

<details>
 <summary>Обяснение: </summary>
 Имаме два вложени цикъла:

- Вътрешният (j) върви от 1 до 3.

- Но когато j == 2, се изпълнява break, което спира само вътрешния цикъл, не и външния.

- Значи за всяко i се отпечатва само когато j = 1, защото при j = 2 цикълът прекъсва.

- За i = 1 → отпечатва 11

- За i = 2 → отпечатва 21

Общо 2 пъти се отпечатва нещо.
</details>

5. Какви числа ще се отпечатат?
```cpp
#include <iostream>
int main() {
    int x = 0;
    do {
        std::cout << x;
        if (x) break;
    } while (x++);
}
```
<details>
 <summary>Обяснение: </summary>
Първо x = 0.

  - Влиза в цикъла (do се изпълнява поне веднъж).

  - Отпечатва 0.

  - Проверява if (x) → if (0) е невярно → не прекъсва.

  - След това x++ увеличава x на 1.

  - Проверява условието while (x++):

  - След първата итерация x вече е 1, така че условието е вярно → влиза отново.

Втори път:

  - Отпечатва 1.

  - Проверява if (x) → if (1) е вярно → break прекъсва цикъла.
</details>
